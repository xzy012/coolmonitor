# 数据库升级安全性说明

## 概述

酷监控系统采用安全的数据库升级机制，确保用户数据在系统更新时不会丢失。所有数据库结构变更都通过版本化的升级脚本进行，并包含完整的数据备份机制。

## 重要文件

### database-upgrader.ts
- **位置**: `src/lib/database-upgrader.ts`
- **作用**: 数据库升级核心文件，包含所有版本升级脚本
- **修改说明**: 当需要添加新的数据库功能时，必须在此文件中添加新的版本升级脚本
- **版本管理**: 每个新功能对应一个版本号，按顺序递增

#### 添加新版本的步骤
1. 在 `DB_VERSIONS` 数组中添加新的版本对象
2. 设置正确的 `version` 号（按顺序递增）
3. 定义 `name`（版本名称）
4. 设置 `requiredTables`（该版本需要的表）
5. 实现 `check` 函数（检查该版本是否已完成）
6. 实现 `upgrade` 函数（执行升级脚本）
7. 更新本文档的版本历史

## 升级机制

### 1. 自动备份
- 每次升级前自动备份数据库
- 备份文件保存在 `data/backup/` 目录
- 保留最近3个备份文件，自动清理旧备份

### 2. 版本化升级
- 每个数据库版本都有明确的升级脚本
- 升级过程是增量的，只添加新功能，不删除现有数据
- 支持从任意版本升级到最新版本

### 3. 安全检查
- 升级前检查所有必要表是否存在
- 升级后验证表结构是否正确
- 如果升级失败，系统会回滚到安全状态

## 升级流程

### 自动升级
1. 系统启动时自动检测数据库版本
2. 如果版本低于最新版本，自动执行升级
3. 升级过程中创建备份
4. 升级完成后更新版本号

### 手动升级（仅用于问题排查）
```bash
# 强制升级（仅用于问题排查）
touch data/.db-upgrade-needed
# 然后重启应用
```

## 数据安全保证

### 1. 数据完整性
- 所有升级脚本都经过测试
- 升级过程是事务性的
- 失败时会自动回滚

### 2. 备份机制
- 升级前自动备份
- 备份文件包含时间戳
- 保留多个历史备份

### 3. 兼容性
- 新版本向后兼容
- 不会删除现有数据
- 只添加新功能，不修改现有结构

## 生产环境升级

### 升级前准备
1. 确保有足够的磁盘空间（至少是数据库大小的3倍）
2. 建议在低峰期进行升级
3. 确保有完整的备份

### 升级过程
1. 停止应用服务
2. 备份数据库文件（可选，系统会自动备份）
3. 更新应用代码
4. 启动应用，系统会自动升级数据库
5. 验证功能正常

### 升级后验证
1. 检查所有功能是否正常
2. 验证数据完整性
3. 确认新功能可用

## 故障恢复

### 如果升级失败
1. 检查日志文件了解错误原因
2. 恢复最新的备份文件
3. 联系技术支持

### 备份文件位置
- 自动备份：`data/backup/coolmonitor.db.[timestamp]`
- 手动备份：建议备份整个 `data/` 目录

## 注意事项

### 1. 版本兼容性
- 确保应用版本与数据库版本匹配
- 不要跳过版本升级
- 按顺序执行所有升级脚本

### 2. 性能考虑
- 大数据库升级可能需要较长时间
- 升级期间避免其他操作
- 建议在维护窗口进行升级

### 3. 监控
- 升级过程中监控系统资源
- 检查日志文件确认升级成功
- 验证所有功能正常工作



## 技术支持

如果在升级过程中遇到问题：
1. 检查应用日志
2. 查看数据库备份文件
3. 联系技术支持团队
